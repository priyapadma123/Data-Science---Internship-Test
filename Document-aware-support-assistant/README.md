# CloudSync Support Assistant (Document-Aware RAG with Memory)

A support chatbot that answers questions from a knowledge-base document,
remembers the conversation, avoids repeating itself, handles topic switches,
and cites the exact section it's drawing from.

## Project structure

```
rag_assistant/
├── documents/
│   └── support_manual.md     # the knowledge base (sample CloudSync support manual)
├── ingest.py                 # splits the doc into section-level chunks + embeds them
├── retriever.py               # loads the embeddings, does top-k similarity search
├── memory.py                  # tracks chat history + which sections were already shown
├── assistant.py               # ties retrieval + memory + LLM together each turn
├── demo_conversation.py       # scripted 11-turn demo showing all 4 required behaviors
├── app.py                     # Streamlit chat UI
├── index/                     # generated by ingest.py (embeddings.npy + metadata.json)
└── requirements.txt
```

## How it works (the pipeline)

1. **ingest.py** reads `documents/support_manual.md`, splits it on `###`
   subsection headers (e.g. "2.3 Cancelling a Subscription"), and embeds each
   section separately using a free local model (`all-MiniLM-L6-v2` via
   `sentence-transformers` — no API key, runs on your machine). Saves
   embeddings + metadata to `index/`.

2. **retriever.py** loads that index and, given a question, embeds it the
   same way and returns the top-k most similar sections by cosine similarity.

3. **memory.py** holds the running chat history (so the model has full
   context of the conversation) and a set of "sections already shown to the
   user this session" — this is what powers the no-repeat behavior.

4. **assistant.py** runs each turn: retrieve → check which retrieved
   sections were already covered → build a labeled context block → send it
   all to the model with a system prompt that requires inline citations like
   `(Section 2.3 - Cancelling a Subscription)` → update memory.

5. **demo_conversation.py** runs 11 scripted turns covering: a topic
   (billing), follow-ups on the same topic, a deliberate repeat to test
   no-repeat behavior, a topic switch to troubleshooting, a switch back to
   billing (testing memory persists across switches), and two more topic
   switches (privacy, API).

## Setup — completely free

```bash
pip install -r requirements.txt
```

Get a free Groq API key (no credit card, see console.groq.com), then create
a `.env` file in this folder:
```
GROQ_API_KEY=gsk_your_key_here
```

**Build the index first** (only needed once, or after editing the document):
```bash
python3 ingest.py
```
This downloads the embedding model the first time (small, ~80MB, one-time).

## Run

Scripted demo (proves all 4 required behaviors in one go):
```bash
python3 demo_conversation.py
```

Interactive chat UI:
```bash
streamlit run app.py
```
